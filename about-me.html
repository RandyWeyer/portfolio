Coding as a lifestyle

In May of 2016, I watched a documentary going over the basic uses of algorithms and how they have changed dramatically in the last 10-20 years. This is how I learned about the classic computer science problem of the Travelling Salesman. For those of you that aren't familiar with it, it's deceptive simple to describe but there are current no known perfect solutions.

The problem is, given a set number of points (generally on a 2-dimensional plane), what is the shortest path between all the points. Sometimes people see this as a combinatorial problem and try to optimize sorting and evaluating distances in each path. Others may see it as a geometric problem that requires proofs so that a computer could always give the optimal path based on coordinates or "weights" of each path.

I had a theory that worked on paper but I needed the precision, speed and adaptability of a computer program to attempt to prove that my theory worked. I started learning Visual Basic and etched out a rough version of that. After that I decided to learn Java and translated the basic algorithm into the Java language. Through this I independently recreated Prim's Algorithm for Minimum Spanning Trees.
